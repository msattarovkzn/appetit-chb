# Лендинг подписки «Аппетит» — Челябинск

Этот проект — клон казанского лендинга (`Ленгинг на подписку МAX`, репозиторий
`appetit-food`), адаптированный под Челябинск. Структура, стили и визуал
(`hero-top.png`, `новый баннер/топ 2.png`, `favicon/`) — общие для всех
городов, без правок.

## Что уже сделано

* `index.html` переписан на структуру казанского лендинга (Hero + анимированные
  кнопки MAX/Telegram + визуальный баннер с overlay-зонами + SEO).
* Город заменён везде: title, description, OG, Twitter, schema.org,
  H1, alt-тексты, footer.
* Ссылки на подписку — рабочие, найдены в `social-popup-widget-chelyabinsk/config.js`:
  * MAX: `https://max.ru/id166101157610_6_bot`
  * Telegram: `https://t.me/AppetitChelyabinck_bot`
* UTM-метки сохранены по той же схеме, что в Казани
  (`utm_campaign=appetit_subscribe`, `utm_medium=hero|middle|bottom`).
* Удалён `btn-bottom.png` — рудимент старой версии кнопки, в новой структуре
  не используется. `banner-bottom-clean.png` оставлен — используется как
  OG-картинка для шаринга.

## Что нужно доделать перед запуском

1. **Домен.** В `index.html` есть плейсхолдеры `https://ЗАМЕНИТЬ-ДОМЕН/`
   (canonical, og:url, og:image, twitter:image, schema.org `url`/`image`) —
   заменить на реальный адрес после деплоя.
2. **Яндекс.Метрика.** Счётчик пока не подключён.
   * `109190808` — счётчик казанского сайта, использовать для Челябинска нельзя
     (испортит статистику обоих городов).
   * `101474146` — это отдельный счётчик попап-виджета (общий для Казани и
     Челябинска), тоже не подходит для лендинга.
   * Нужно завести **новый отдельный счётчик** в Яндекс.Метрике под этот
     лендинг, вставить код счётчика в `<head>` (по образцу казанского
     `index.html`) и пересоздать 6 целей:
     `max_click_hero`, `telegram_click_hero`, `max_click_mid`,
     `telegram_click_mid`, `max_click_bottom`, `telegram_click_bottom`.
   * После этого в конце `index.html` заменить
     `var METRIKA_COUNTER_ID = null;` на реальный ID счётчика.
3. **tgtrack-скрипт MAX — пропущено.** В казанской версии подключён скрипт
   `max.tgtrack.ru/.../?linkID=...&chat=appetit_dostavka&counterID=109190808`.
   Для Челябинска канал к tgtrack не подключался (alias/linkID не выдавался),
   подписка работает через прямую ссылку на бота без этого скрипта — решили
   не блокироваться на нём. Можно подключить позже, если подключите канал
   к tgtrack: понадобятся свои `linkID` и `chat`, а `counterID` — это ID
   счётчика Метрики (см. пункт 2).
4. **yandex-verification.** В казанской версии есть meta-тег верификации
   Яндекс.Вебмастера — для нового домена потребуется отдельная верификация
   (новый код, плюс файл `yandex_XXXXXXXX.html` в корне).
5. **Деплой.** Когда домен определится — обновить `vercel.json` при
   необходимости и задеплоить (репозиторий уже привязан к GitHub:
   `msattarovkzn/appetit-chb`).
